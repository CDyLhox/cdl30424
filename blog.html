in the meanwhile here you have some gifs

anyways, if im telling the truth i dont know much about what im doing when it comes to 3d or scanning technology.
i know what i like and what i dont like and i know enough in blender to make some projects to explain myself in a 3d context.
for example i always tended to use *.fbx as model exports instaid of *.obj because the app i like to use scaniverse says thats a widely used model export and also i like that it is only one file compared to obj which is 2 or 3 files depending on the program. Lately though ive been exporting *.ply models from scaniverse also because my good friend Lucas explained to me that you can also export point clouds from the app. Before knowing this whenever id need a point cloud id make one out of my model via the blender node system putting unnecessary strain on my poor M1 chip.

so not being a trained 3d artist also makes my renders WAAAY complicated and very unoptimized, this is also another reason why i want to use three.js instaid of using my usual P5. because i feel like threeJS is better optimized for 3d (or atleast id hope so) and also their material system is way nicer, but im still figuring out the asynchronic rendering and data flow.
i feel i should make a list of programs i like to use but maybe i wont do so within this blog post here..


anyways if youre working in blender blenderKit is in my opinion an essential addon to have because of the endless world of prefabs(prefabricated objects) and materials. and speaking of blender addons Sapling Tree Gen is also a great addition to your toolbelt because making trees is hard and this can make pretty much any tree you want for you. the issue i found was that my trees became very loadbaring for my poor m1 chip. i am also now finding out that this sapling tree gen figure im describing is a built in blender feature so themoreyouknow i guess.
then also while im at it for a shoddy javascript experience with 3d you can work pretty well in p5Js but if you have the time to really learn it then probably threeJs is a better option for 3d in specefic. theyve got some super cool examples on their website and when i get the time i hope to figure out how this works because to me it seems like a beautiful medium for audio/visual worldbuilding. anyways, i think ive said about all i really know about this topic and maybe even a little bit more than that. right now its 23:44 on tuesday 17th of december 2024 and i should really be working on an additive synthesizer im programming in c++ for a school project so that might be what im going to do now who knows really.. if you havent you should check out my Friends.html page some of my friends have some very cool websites also to check out #bringbacktheWWW or something and also also you should click the (prefabricated objects) link for some nice music byebye.
